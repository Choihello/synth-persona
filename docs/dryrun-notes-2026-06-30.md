# key-free persona dry-run 메모 — 2026-06-30

> **목적**: Haiku 실측 전에 입력 품질·실험 설계를 점검한다. **외부 LLM 호출 없음.** 아래 §4 질적 리뷰는 *사람(reviewer)이 페르소나를 읽고 세운 사전 가설*이며 **실제 LLM 응답이 아니다** — 데이터로 인용 금지. 모든 응답 숫자는 mock 기반 **synthetic panel response**다.
>
> 재현: census 스냅샷 → `synthesizePopulation` → `sampleForSimulation(pop, N, seed=42)` → (mock) `simulate`/`aggregate`/`assessReliability`. 공개 API만 사용.

## 2) dry-run 질문 → provider에 전달되는 실제 prompt

`choices`가 있으면 `buildPrompt`가 아래처럼 합성해 provider(실측 시 Claude)에 전달한다:

```
월 9900원 신선식품 새벽배송 구독, 쓸 의향?

아래 선택지 중 정확히 하나만 고르고, 그 선택지 문구를 그대로 답에 포함하세요:
- 쓴다
- 안쓴다
```

(나머지 2개도 동일 구조: "신뢰한다/신뢰안한다", "끌린다/안끌린다".)

- ⚠️ **choices 문구는 모델이 *그대로* 답에 포함해야 `matchChoice`가 잡는다.** "신뢰안한다"·"안끌린다"는 어색해서, Haiku가 "신뢰하지 않는다"로 답하면 부분문자열 매칭 실패 → `missing`. 자연스러운 choices + 실측 시 `missing` 비율 모니터링 필요.

## 3) mock census 실행 (N=12, seed=42, "9900원 새벽배송")

- 전체 신호: **🔴 split** (분산 0.81), 응답 12건
- 응답 분포(synthetic panel response): `쓴다=3, 안쓴다=9` ← mock 결정적 결과, 실제 반응 아님
- 신뢰성 카드: 1층 🟢 MAE 0.0000 / 2층 성·연령·지역=matched(높음), 혼인·가구원수=conditioned(중간, 가구원수에 bridge note) / 3층 not-measured / 가드레일: synthetic panel response·가격 부적합(소득·직업·자녀 축 없음)
- ⚠️ **2층 카드는 표본 의존적**: 이 표본엔 18세 미만이 없어 혼인=conditioned(중간). `reliability:demo`(N=200)처럼 15~17세가 섞이면 혼인=inferred(낮음)로 worst-wins 강등된다.

## 4) 질적 사전 리뷰 (⚠️ 사람의 가설 — 실측 아님, 데이터 인용 금지)

페르소나 5~8명을 읽고 "이 질문에 응답 차이가 날 법한 결"을 사람이 추정한 것:

- **9900원 새벽배송**: 1인·젊은 수도권(s6)과 고령 2인 비수도권(s2, s10)에서 갈릴 법. 단 가격 민감도는 **소득 축이 없어** 추정 불가 — 방향(누가 관심)까지만.
- **AI 예산 추천 신뢰**: 고령(s2, s10)에서 신뢰 낮고 젊은 층(s1, s6)에서 높을 법한 *신뢰/수용* 질문. 가격 무관 → 현재 축(연령·지역·혼인)과 궁합 좋음.
- **지역 소상공인 묶음배송**: 가구 규모(s5 3명 vs s6 1명)와 지역(수/비)으로 갈릴 *선호* 질문. 현재 축으로 세그먼트 해석 가능.

→ **이건 실측 결과가 아니라 "무엇을 물어봐야 차이가 보일지"의 설계 점검이다.**

## 5) 결론

### 실측 후보로 좋아 보이는 질문
1. **AI 예산 추천 신뢰?** — 가격 무관·신뢰/수용 축, 현재 인구 속성(연령·지역·혼인)으로 해석 가능. **1순위.**
2. **지역 소상공인 묶음배송 끌리나?** — 선호/채택, 가구·지역으로 세그먼트. **2순위.**
3. **9900원 새벽배송?** — 방향(관심 세그먼트)엔 유효하나 **가격 결론은 소득축 결핍으로 저신뢰** (가드레일이 이미 경고). 가격 질문보다 "관심 여부"로 재프레이밍 권장.

### 부족해 보이는 세그먼트/속성
- **소득** — 가격탄력성·구매력 판단 불가 (가드레일이 정직히 경고 중). 가격 질문의 최대 제약.
- **직업/라이프스타일** — 새벽배송·구독 같은 생활패턴 질문의 결을 더 가를 축.
- **자녀 유무** — 가구원수는 있으나 자녀 구성은 없음. 가족 타깃 상품 해석 한계.
- 희소 세그먼트(사별·이혼, 5·6인가구)는 소표본에서 과대/과소 표집 → 실측은 N 키우기.

### Haiku 실측 시 체크리스트
- [ ] **choices 자연스럽게** — 모델이 문구를 그대로 답할 수 있게. 실측 후 `missing`(매칭 실패) 비율 확인.
- [ ] **N ≥ 100** — 세그먼트 안정화 + 희소 세그먼트 표집 편차 완화.
- [ ] **전원 실패 처리 확인** — 키 오류/한도 시 `aggregate`가 throw해 false consensus 안 만드는지(이미 구현).
- [ ] **3층(응답 신뢰도) 채우기** — `censusShareRunner`로 robustness 실측 + probes(자기일관성·예스맨·평균회귀) → `responseConsistency` placeholder 교체 (issue #3).
- [ ] **비결정성 대비** — Haiku는 mock과 달리 비결정적. VCR 녹화로 재현성 확보 권장.
- [ ] **2층 카드 해석** — 혼인/가구원수 confidence가 표본의 <18세 포함 여부로 medium↔low 변동함을 인지.
- [ ] **가드레일 유지** — 가격 질문엔 소득축 결핍 경고가 그대로 노출되는지.

### 판단: Haiku 실측으로 넘어가도 되는가?
**파이프라인·입력 품질·가드레일은 실측 준비 완료.** 다만 (a) choices 문구를 자연스럽게 다듬고, (b) 1순위로 "AI 예산 추천 신뢰?"처럼 가격 무관 질문부터 시작하는 게 안전. 가격/구매력 질문은 소득축이 들어오기 전까지 "방향 탐색"으로만. 실측 착수 자체는 `ANTHROPIC_API_KEY`만 있으면 가능(issue #2).
