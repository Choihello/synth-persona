## 2. Haiku 실측 착수 조건

> ⚠️ **실제 Haiku 호출은 사용자 승인 + 키 준비 전까지 금지.** 첫 실측은 **시장 판단이 아니라 pipeline smoke test**로 라벨링한다.

- **`ANTHROPIC_API_KEY` 필요** — `.env`(gitignored)의 빈 `ANTHROPIC_API_KEY=`에 실제 키. 현재 없음.
- 파이프라인은 준비 완료 — `--source census`에서 `--mock`만 빼면 `ClaudeProvider`로 동작. `LoggingProvider`로 raw/latency/error 캡처.
- **첫 실측 질문(저위험)**: "AI가 장보기 예산을 자동 추천해준다면 신뢰할 수 있나요?" (`신뢰할 수 있다`/`신뢰하기 어렵다`). 가격/구매의향은 **소득축 결핍**으로 저신뢰 — 방향 탐색으로만.
- 체크리스트: choices 자연스럽게, missing rate 모니터링(어미 변형 미매칭), 전원 실패 시 `aggregate` throw(false consensus 차단, 구현됨), 비결정성은 `RecordedProvider`(VCR) 녹화로 재현성 확보.

## 4. 설계 불변식 (유지 — 깨지 말 것)

- 숫자 = **synthetic panel response**, "구매율/시장 예측" 단정 금지. 불확실성을 consensus처럼 포장 금지(응답 0건→`aggregate` throw).
- 1층(구성)과 2층(LLM 응답) 출력 분리. **"완전 5-way joint" 비주장**(3축 matched-core + 2 conditioned).
- `Persona.weight` · provenance(matched/conditioned/inferred) · `householder_age_as_proxy` bridge · suppressed(`X`)≠structural zero(`-`).
- 런타임 외부 의존성 **`@anthropic-ai/sdk` 단 하나** — 추가 금지(`node:crypto` 등 빌트인은 허용).
